# EduBridge University — Backend

Платформа помогает студентам поступить в зарубежные университеты: подбирает подходящие вузы на основе данных студента, помогает с документами и ведёт по пути подачи документов поэтапно.

---

## Технологический стек

| Компонент        | Технология                              |
|------------------|-----------------------------------------|
| REST API         | Python (FastAPI) / Node.js (на усмотрение команды) |
| База данных      | PostgreSQL                              |
| Файловое хранилище | MinIO                                 |
| Контейнеризация  | Docker + Docker Compose                 |
| Аутентификация   | JWT Access Token + Refresh Token        |
| Email-уведомления | SMTP (SendGrid или любой провайдер)    |

---

## Быстрый старт

### Требования

- Docker >= 24
- Docker Compose >= 2.20

### Запуск

```bash
# Клонировать репозиторий
git clone <repo-url>
cd edubridge-university

# Скопировать и заполнить переменные окружения
cp .env.example .env

# Запустить все сервисы
docker compose up -d --build

# Проверить состояние
docker compose ps
```

API будет доступен по адресу: `http://localhost:8000`

Документация (Swagger / OpenAPI): `http://localhost:8000/docs`

MinIO Console: `http://localhost:9001`

## Переменные окружения

| Переменная            | Описание                          | Пример                     |
|-----------------------|-----------------------------------|----------------------------|
| `DATABASE_URL`        | Строка подключения к PostgreSQL   | `postgresql://user:pass@db:5432/edubridge` |
| `JWT_SECRET`          | Секрет для подписи JWT            | `supersecretkey`           |
| `JWT_ACCESS_EXPIRE`   | Время жизни Access Token (мин)    | `30`                       |
| `JWT_REFRESH_EXPIRE`  | Время жизни Refresh Token (дней)  | `14`                       |
| `MINIO_ENDPOINT`      | Адрес MinIO                       | `minio:9000`               |
| `MINIO_ACCESS_KEY`    | Access Key MinIO                  | `minioadmin`               |
| `MINIO_SECRET_KEY`    | Secret Key MinIO                  | `minioadmin`               |
| `SMTP_HOST`           | SMTP сервер                       | `smtp.sendgrid.net`        |
| `SMTP_PORT`           | SMTP порт                         | `587`                      |
| `SMTP_USER`           | SMTP логин                        | `apikey`                   |
| `SMTP_PASSWORD`       | SMTP пароль / API-ключ            | `SG.xxx`                   |
| `EMAIL_FROM`          | Адрес отправителя                 | `no-reply@edubridge.com`   |

## Роли и доступ

| Роль      | Описание                                             |
|-----------|------------------------------------------------------|
| `admin`   | Полный доступ ко всем ресурсам                       |
| `student` | Доступ только к своим данным и публичному контенту   |

Регистрация — **только через администратора**. Студент оставляет заявку (lead), администратор создаёт аккаунт — логин и пароль генерируются автоматически и отправляются студенту на email.

### Формат автогенерируемого логина

```
<имя_латиницей><4 случайные цифры из номера телефона>
Пример: ivan4782
```

## Основные модули API

| Модуль           | Базовый путь              | Описание                                      |
|------------------|---------------------------|-----------------------------------------------|
| Аутентификация   | `/api/auth`               | Вход, выход, обновление токена, сброс пароля  |
| Заявки (Leads)   | `/api/leads`              | Заявки от незарегистрированных студентов      |
| Пользователи     | `/api/users`              | Профили студентов                             |
| Университеты     | `/api/universities`       | Каталог вузов, поиск, подбор, избранное       |
| Этапы            | `/api/universities/{id}/stages` | Этапы поступления для конкретного вуза  |
| Прогресс         | `/api/training`           | Прохождение этапов студентом                  |
| Уведомления      | `/api/notifications`      | История и управление уведомлениями            |
| Уроки            | `/api/lessons`            | Текстовые и видео-уроки                       |
| Кейсы            | `/api/cases`              | Истории успешных поступлений                  |
| Файлы            | `/api/files`              | Загрузка файлов, получение pre-signed URL     |
| Дашборд          | `/api/admin/dashboard`    | Сводная статистика для администратора         |

Полная спецификация эндпоинтов — см. [STRUCTURE.md](STRUCTURE.md).

## Безопасность

- Все эндпоинты (кроме `POST /api/leads`) требуют `Authorization: Bearer <token>`
- Файлы выдаются только через **pre-signed URL** с ограниченным временем жизни
- Прямое скачивание файлов через API запрещено
- Аккаунты со статусом `archived` и `enrolled` не получают JWT
- CORS настроен под фронтенд-домен
